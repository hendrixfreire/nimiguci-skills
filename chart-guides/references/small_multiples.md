# Small Multiples Pattern

Use this guide when one chart would be overloaded and repeated aligned mini-charts will improve comparison.

> Always apply global rules from [master_style.md](master_style.md) first.

## When to use

Use when:
- many series would clutter a single line or bar chart
- repeated category or regional comparisons need the same visual grammar
- the audience benefits from scanning differences across panels

Common cases:
- one line chart per channel
- one bar chart per region
- one histogram per segment
- one scatter panel per category

## When not to use

- when panels become too small to read
- when over-faceting creates a wall of tiny charts without clear priority
- when one integrated chart would be simpler and still legible

## Core rules

1. Keep axes aligned across panels when direct comparison matters.
2. Keep scales consistent unless there is a strong reason not to.
3. Use the same category order, time order, and color meaning across panels.
4. Keep panel titles concise and informative.
5. Reduce decorative elements aggressively.

## Semantic rules

- If scales differ across panels, state it explicitly.
- If one panel is highlighted, explain why.
- Do not create so many panels that none of them is readable at the final size.

## Quick checklist

- [ ] One combined chart would be cluttered
- [ ] Panel count is manageable
- [ ] Shared scale logic is clear
- [ ] Titles and labels remain readable at destination size
