# Comparison Chart Patterns

Use this guide for dot/lollipop charts, bullet charts, and slope charts.

> Always apply global rules from [master_style.md](master_style.md) first.

## Dot / Lollipop charts

### Use when
- ranking categories with a lighter visual than bars
- emphasizing precise position rather than filled area
- category names may be long and horizontal layout helps

### Do not use when
- bar length would be faster for the intended audience
- decorative stems in lollipop charts add more style than clarity

### Style rules
1. Prefer dot plots first; add stems only if they improve scanning.
2. Highlight the key category in purple.
3. Keep all non-highlight marks visually secondary.
4. Sort categories when ranking is the point.

## Bullet charts

### Use when
- comparing actual vs target or benchmark efficiently
- qualitative performance zones add meaning

### Do not use when
- a simple bar with target marker is clearer for the audience
- multiple bullets would become too dense for the available space

### Style rules
1. Keep actual performance dark and prominent.
2. Keep target/reference marker crisp and unmistakable.
3. Keep qualitative bands subtle.
4. Use concise labels and values.

## Slope charts

### Use when
- comparing two points in time or two states
- direction and rank movement matter

### Do not use when
- more than two or three points are needed
- too many categories create line clutter

### Style rules
1. Label both ends directly whenever possible.
2. Keep line count controlled.
3. Highlight the key category or story line in purple.
4. Use neutral lines for non-priority categories.

## Quick checklist

- [ ] This comparison pattern is simpler than a heavier chart family
- [ ] Labels can be read without hunting
- [ ] The emphasized item truly deserves emphasis
- [ ] Ranking/order supports interpretation
